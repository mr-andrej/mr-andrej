### Natixis CIB — Full-Stack Developer · *Sept. 2025 — Present*
- **Designed and built end-to-end** a production monitoring dashboard tracking the health of internal APIs and data providers across the Credit Risks department — delivered as a microservice backed by Oracle DB.
- **Authored and defended an architecture study** for a distributed Redis cache before an architecture committee, securing approval to build and deploy the full-stack solution.
- Built **Kiroshi**, a PowerShell tool for industrializing Control-M jobs: detects existing configurations (technical accounts, server farms, VMs), interactively confirms keep-or-replace, and generates an HTML conformity report — catching anomalies before deployment and preventing production incidents.
- Designed and maintained internal applications from SQL databases to Angular front-ends, contributing to CI/CD practices across the credit risk perimeter.

### Société Générale — Full-Stack Developer · *Sept. 2023 — Sept. 2024*
- **Built comprehensive E2E test scenarios** with Cypress: Business Analysts now validate entire user journeys in one click, instead of manually filling 20+ fields every test cycle.
- Set up the **dedicated CI pipeline** running these tests automatically, producing a video artifact per run — instant diagnosis on failure.
- Feature development, optimization, and UI/UX improvements within the SG Initiatives team, including SG Design System integration.

### Actimage × French Ministry of National Education — Full-Stack Developer · *Sept. 2022 — Aug. 2023*
- **Lead developer of the national platform** used to create medical school exams (EDN & ECOS).
- **Refactored the "random draw" system** assembling randomized exam question groups: load time cut from 30s at the 3rd draw (+5s per additional draw) to a fixed 10–15s regardless of volume — **>80% reduction at scale**.
- Designed a new "point-and-click zone" question type for medical imagery annotation, among dozens of shipped features.

### Okédoc — Full-Stack Developer · *Feb. 2021 — Apr. 2022*
- **Sole developer** of a web application certified as a Medical Device for multiple sclerosis patients, built from scratch around an Azure chatbot — in the pre-LLM era.
- **Designed and built the platform's proprietary authentication system**, replacing Okta — full control over the auth lifecycle and one less third-party dependency (and bill).
- Chose an **embedded SQLite database**: zero administration, trivial backups, minimal footprint — a pragmatic right-sizing call for a certified application.
- Owned the full Azure infrastructure lifecycle, coordinated external third parties, and implemented click/time-on-page tracking to guide product decisions.
